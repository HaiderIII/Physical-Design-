------------------------------------------------------------
SECTION D — PATH CLASSIFICATION
------------------------------------------------------------

10. Timing path categories:
    - IN → FF
    - FF → FF
    - FF → OUT

11. Examples of paths:
    - IN → FF: a/b → U1 → U2 → U3 D
    - FF → FF: none in this netlist (only one sequential element)
    - FF → OUT: U3 Q → y

12. Reasoning:
    - Paths treated differently in STA because timing constraints and setup/hold checks depend on start and end types (input, sequential, output). IN→FF requires input arrival analysis, FF→FF requires sequential launch/capture checks, FF→OUT verifies output timing.
